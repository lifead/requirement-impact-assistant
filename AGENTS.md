# AGENTS.md - оперативные правила проекта

## Назначение проекта

**inventory-api-training** - учебный backend-проект для отработки полного цикла разработки на примере модуля "Склад" (Inventory).

Проект ведется как учебный workflow: от анализа сырой задачи и требований до проектирования, реализации, тестов, ревью и документации.

## Workflow feature

Крупная feature проходит этапы последовательно:

1. Сырая постановка задачи.
2. Первичный предпроектный анализ.
3. Фиксация уточнений и решений.
4. Черновик требований.
5. Ревью требований.
6. Технический проект.
7. Реализация.
8. Проверка и ревью изменений.

## Phase gates

- Нельзя переходить к требованиям без завершенного первичного анализа и зафиксированных уточнений.
- Нельзя переходить к техническому проектированию без утвержденных требований.
- Нельзя переходить к реализации без завершенного технического проекта и явного решения о начале реализации.
- На ранних этапах не фиксируются преждевременные технические решения: БД, структура таблиц, endpoint-ы, архитектурный стиль, фреймворки.

## Правила работы

- **Код не создается до отдельного явного решения.** Решение о начале реализации принимается после завершения анализа, требований и проектирования.
- **Все изменения выполняются маленькими шагами.** Один шаг - одна законченная мысль или операция.
- **Commit выполняется только по явной команде пользователя.** Без команды "commit" изменения не фиксируются.
- **Перед каждым commit необходимо показать `git diff --stat`**, чтобы пользователь видел сводку изменений.
